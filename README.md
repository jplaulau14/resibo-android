# Resibo

> **Resibo** (Filipino for "receipt") — an Android app that gives you the receipts on any social media post you share to it. On-device, offline, multilingual, calibrated.

Hand it a post (text, image, video, audio) via the Android Share Target and it returns a **Note**: sources, reasoning trace, confidence band, and what it couldn't verify — generated by Gemma 4 running fully offline on your phone.

**The agent produces a Note, not a Verdict.** You decide.

## What it does

- Accepts shared posts from any Android app (text · image · video · audio)
- Normalizes input: text extraction, perceptual hashing, Whisper-small for audio, first-frame extraction for video
- Runs a local agent loop: claim decomposition → domain routing → RAG retrieval → reasoning → Note generation
- Uses a **1B model for triage**, **4B model for reasoning** (routed by task complexity and available RAM)
- Returns a structured Note with sources, confidence band, and explicit abstentions
- Caches Notes locally and surfaces community Notes for previously-seen viral posts via perceptual hash matching

## Stack

- **Platform**: Android-only, native Kotlin + Jetpack Compose
- **Runtime**: Google AI Edge + LiteRT
- **Models**: Gemma 4 1B (triage) + Gemma 4 4B (reasoner), quantized
- **Languages supported**: Tagalog, English, Taglish, Cebuano, Bisaya
- **RAG corpus**: PH fact-check archives (Tsek.ph, Vera Files, Rappler), Cochrane/WHO/DOH for health, PSA OpenSTAT for statistics
- **On-device**: no sign-in, no telemetry, encrypted local storage

## Build

Requirements:

- Android Studio Ladybug or later
- JDK 17
- Android SDK 34+
- A physical device with ≥ 6 GB RAM for 4B model inference (Pixel 8 Pro recommended)

```bash
./gradlew assembleDebug
./gradlew installDebug
```
